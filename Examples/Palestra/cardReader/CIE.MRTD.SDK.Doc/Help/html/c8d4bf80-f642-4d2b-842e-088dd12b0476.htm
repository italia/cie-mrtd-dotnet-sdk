File di datiCarta d'Identità Elettronica - MRTD SDK

Questa sezione descrive brevemente i Data Group dei Machine Readable Travel Document (MRTD)

Contenuto

I dati contenuti negli MRTD sono organizzati in file binari chiamati Data Group, ognuni dei quali ha una struttura e un contenuto definito dallo standard ICAO. Vengono elencati di seguito i DG previsti dallo standard ICAO. Tutti i Data Group e i file dati sono codificati in formato binario ASN.1. Alcuni DG non sono elencati poiché non sono utilizzati nella Carta d'Identità Elettronica

  • DG1

    Il DG1 contiene la stringa MRZ stampata sul documento

  • DG2

    Il DG2 contiene l'elemento biometrico primario, cioè la foto del titolare. La foto è codificata in formato JPEG2000 secondo lo standard ISO/IEC 19794-5

  • DG3

    Il DG3 contiene l'elemento biometrico secondario, cioè le impronte digitali del titolare. Le impronte sono memorizzate in formato WSQ secondo lo standard ISO/IEC 19794-4

  • DG7

    Il DG7 contiene l'immagine della firma autografa del titolare. L'immagine è codificata in formato JPEG2000

     Note

    Il DG7 non è presente nella Carta d'Identità Elettronica

  • DG11

    Il DG11 contiene dati relativi al titolare del documento, come l'indirizzo di residenza, la data e il luogo di nascita, il codice fiscale

  • DG12

    Il DG12 contiene dati relativi al documento, come il luogo e la data di rilascio e l'autorità emittente

  • DG14

    Il DG14 contiene la chiave pubblica e i parametri di dominio Diffie Hellman utilizzati in fase di Chip Authentication

File di servizio

Oltre ai DG sono presenti altri file il cui contenuto è legato ai processi di autenticazione e verifica del documento

  • EF.COM

    L'EF.COM contiene l'elenco dei DG memorizzati nell'MRTD

  • EF.SOD

    L'EF.SOD contiene la firma digitale degli hash dei DG effettuata tramite il certificato di Document Signer e verificata in fase di Passive Authentication

  • EF.CVCA

    L'EF.CVCA contiene il nome della CVCA la cui chiave pubblica è memorizzata nel documento e da cui deve essere emesso il certificato di Inspection System in fase di Terminal Authentication
